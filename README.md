# r
Rust no more; "TRust programming language"

The TRust programming language, for the user; Bringing back the object oriented/imperative programming paradigm, to program like its the 90s again (albeit a modern take on OOP)
and experimenting with how to code in a more natural and beautiful way (without this like "integrate and depend with everything ever" mindset)

There's a version of the references that I'm still thinking about—how we can do references differently; Alongside the object-oriented/imperative models. I can see a world where these two things exist in such a way, that its like: duh, why didnt anyone think of it before.

Traits are like Interfaces, atleast that is the direction I would lean towards. Things like struct embeddings would make it easier to mix and match behaviors without rigid hierarchies, and I think we are leaning towards something like this... It would also be important to change the idea of namespaces, splitting it into different aspects (I dont like when modules are solving the part of name conflicts and such)

It should be like a freeform object language. So alot of these things depends on how well they play into the idea of a freeform object space. Many things would be kindve different, but in the sense that it would be like, a simplification. There are things in Rust that are okay in terms of the sentiment, but that I feel like are poor in execution, almost like a team of a million people were fighting over a certain feature, and they ended up putting them all in, appeasing the voice of every human on Earth. Rust is tryina be, and do, three million things in 5 lines of code, and its really about keeping it simple in alot those fundamnetal areas. 

I'd like it to stay closer to a C/Python syntax in regards to closures and stuff which are now in a lot of languages (I dont know that has to be that many things in a language anyway, especially when its still in the early stages, alas we're just spitballing here) That is to say, if things are going to emulate a C/Python sortve domain of syntax, then it should get closer to it in terms of similitude. The idea is of course to make something more beautiful, with all the same stuff that's now important to have in a language (maybe omitting some it) those are things to think about.
