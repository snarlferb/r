# r
Rust no more; "TRust programming language"

The TRust programming language, for the user; Bringing back the object oriented/imperative programming paradigm, to program like its the 90s again (albeit a modern take on OOP)
and experimenting with how to code in a more natural and beautiful way, integrating a version of the references, borrowing, and lifetimes alongside the object-oriented/imperative models.

That way the Rust debate can just stop; TRust./

Unlike traditional ownership-based systems, TRust explores a hybrid approach where objects can be explicitly owned, borrowed, or referenced with well-defined lifetimes,
and it would also be important to change how namespaces are expressed (possibly with a macro system or something is my most recent idea)

Note: This would NOT be a new age metaprogramming language, as its suppose to be returning to the old style of things, but in a more modern style.

Im also open to making a compiler that is able to act as the "create-a-language" program, that makes it easier for beginners to figure things out on their own,
however im currently working on something else *(that I call a Keepsake A Parser Program)*, that sortve-kindve has a similar notion, and in a way that isnt about
"a new way to make compilers and subsequent languages", and is more like "how to learn what your doing as you generate the thing" -type've thing.

I basically have too many ideas, and so thats what this is: the very edge of the pool w/ all my ideas oozing out.
